# Правила командной работы (Team Guide)

## Ветки и PR

- Рабочая ветка — **dev**; задача — от **dev**, формат: `feature/<short-name>`, `fix/<short-name>`.
- PR направляется в **dev**; нужен минимум один апрув; линт и тесты должны проходить.
- **main** обновляется только через **dev**.

## Code Review

- Перед PR: линтер пройден, код по SOLID/DRY/KISS, без «кода на будущее» и мусорных файлов.
- За спринт — один PR на ревью ментору; для зачёта — два апрува на задачи повышенной сложности.

## Инструменты

- Lefthook — пре-коммит проверки; в исключительных случаях — `--no-verify` (не злоупотреблять).
- CI (GitHub Actions) — линт и тесты; merge при зелёных проверках.
